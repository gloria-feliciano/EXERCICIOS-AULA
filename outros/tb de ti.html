Topologias de rede

O que é topologia de rede?

A topologia de rede é a forma como os dispositivos (computadores, servidores, roteadores, switches, etc.) estão interconectados dentro de um ambiente de TI. Ela define como os dados circulam e influenciam diretamente o desempenho, a segurança e a escalabilidade da rede.

Principais tipos

Barramento- Todos os dispositivos compartilham o mesmo meio de comunicação (cabo principal). Simples, mas suscetível a falhas e congestionamento.
Anel- Os dispositivos são conectados em um circuito fechado. Tem bom controle de tráfego, mas se um nó falha, a rede pode parar.
Malha– Cada dispositivo pode se conectar a vários outros. Alta redundância e segurança, mas custo elevado.
Estrela– Todos os dispositivos se conectam a um switch central. Melhor controle e gerenciamento, além de ser mais fácil de escalar.
Híbrida– Combina diferentes topologias para melhor desempenho e segurança.

melhor topologia para uma empresa de médio porte

A topologia em estrela é a mais recomendada para uma empresa de médio porte que deseja priorizar segurança e desempenho.
Segurança: Como todo o tráfego passa pelo switch central, é mais fácil implementar medidas de segurança como VLANs, firewalls e monitoramento.
Gerenciamento Simples: Permite controle centralizado do tráfego e configuração eficiente.
Escalabilidade: Novos dispositivos podem ser adicionados sem afetar o desempenho.
Menor impacto de falhas: Se um dispositivo falhar, não compromete toda a rede, diferentemente de um anel ou barramento.

Reforçando a segurança

Segmentação com VLANs para isolar setores (RH, TI, atendimento).
Firewall e IDS/IPS para monitoramento e bloqueio de tráfego suspeito.
Autenticação Multifator (MFA) para garantir acesso seguro.
Criptografia de dados para proteger informações sensíveis.
Implementação do modelo Zero Trust, onde nenhum dispositivo ou usuário é automaticamente confiável.

O que é VLAN?

Uma VLAN é um grupo lógico de locais de trabalho, servidores e dispositivos de rede que parecem estar na mesma rede de área local (LAN), apesar da distribuição geográfica. Resumindo, o hardware nas mesmas VLANs permite que o tráfego entre os equipamentos seja separado e mais seguro.


Porta de acesso: Uma porta de acesso transporta o tráfego para apenas uma VLAN. As portas de acesso são geralmente chamadas de porta não marcada, pois há apenas uma VLAN nessa porta e o tráfego pode ser transmitido sem marcas.
Porta de tronco: Uma porta em um switch que transporta tráfego para mais de uma VLAN. As portas de tronco são frequentemente chamadas de portas marcadas, pois há mais de uma VLAN nessa porta e o tráfego para todas as VLANs, exceto uma, precisa ser marcado.
VLAN nativo: A única VLAN em uma porta de tronco que não recebe uma marca. Qualquer tráfego que não tenha marca será enviado para a VLAN nativa. É por isso que os dois lados de um tronco precisam ter a mesma VLAN nativa, ou o tráfego não irá para o local correto.
